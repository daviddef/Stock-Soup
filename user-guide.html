Skill 2 — The Scoring Model (0 to 7+)

Every candidate gets a Base Score (1–5) based on catalyst strength, then Bonuses and Penalties adjust it. Minimum score of 3.5 required to become a pick.

Base Score — Catalyst Strength

5
Major catalyst TODAY — stock hasn't moved yet
Best entry: ASX announcement just filed, deal signed this morning, upgrade just published. Market hasn't priced it in.
4
Strong catalyst — upgrade, deal, or earnings beat
Fresh analyst upgrade, new contract announced, earnings above expectations. Confirmed, not speculative.
3
Volume-backed move with supporting news
Unusual volume spike AND a reason for it — sector news, macro tailwind, small ANN. Both signals together.
2
News only — no volume confirmation
Interesting catalyst but nobody is buying yet. Wait for volume to confirm before entering.
1
Volume only — no catalyst found
Something is happening but we can't explain it. Possible pre-announcement leak. Flag as ❓ INVESTIGATE.

Technical Indicators — Plain Language

RSI (Relative Strength Index)
Measures momentum on a 0–100 scale. Think of it as a speedometer — how fast is the stock moving?
40–65 rising ✓ >70 overbought <30 falling
SMA 50 (50-day Average)
The average share price over the past 50 days. Being above it means the medium-term trend is up.
Price above ✓ Price below ✗
SMA 200 (200-day Average)
The long-term average. Above = confirmed uptrend. Below = bear territory. The "golden cross" (SMA50 crossing SMA200 upward) is a major bull signal.
Above = uptrend ✓ Below = bear zone
Supertrend
A single UP or DOWN signal based on price + volatility bands. Simpler than RSI — it just tells you the trend direction.
UP + ADX>25 ✓ DOWN = avoid
ADX (Average Directional Index)
Measures trend strength, not direction. A weak trend in either direction means choppy trading — hard to profit from.
>25 strong trend <20 = no trend
Volume Ratio
Today's volume vs the 20-day average. 3× means 3 times the normal amount of shares are changing hands — something is happening.
>3× unusual ✓ <0.5× quiet

How Entry / Target / Stop are set

FieldHow it's calculatedWhy it matters
Entry ZoneCurrent price ±1–2% (limit order range)Set a limit order — never chase with a market order

The Three Pick Timeframes

⚡
DAY Pick
Same session or next open
Fast-moving catalyst: short squeeze, gap-and-go, pre-market announcement. Set a tight stop. Exit by end of session if not at target.
📅
WEEK Pick
1–5 trading days
Analyst upgrade, upcoming earnings, deal announced but not fully priced yet. Hold through the week and re-assess Friday.
📈
LONG Pick
Weeks to months
Structural thesis: sector rotation, undervalued vs analyst consensus, AI/commodity megatrend. Patient entry, wider stop, hold through noise.

Under the Hood — Skill Architecture

The system uses four modular skills. Each handles a distinct phase of the pipeline and can be run or re-run independently.

🔍 asx-global-discovery
Phase 3 — Discover: 15 homepage scrapes + 35 keyword searches → candidate pool. Always runs first. Outputs ranked table + sector heat map that feeds directly into scoring. ~245 lines.
📊 hourly-global-stock-picks
Phases 2, 4, 5, 6, 7: Geopolitical briefing → price data (EODHD + Finnhub + smallcaps) → enrichment (AV/Mediastack/Twelve Data — daily cache) → tiered news feeds → scoring → pick selection → report writing. ~1,735 lines · report structure in report-template.md.
📬 publish-report
Phase 8 — Publish: Reads /tmp/email-data.json → email-builder.py fills email-template.html → Gmail draft → publish-report.py pushes HTML to GitHub Pages → writes watchlist-status.json → SMS (paused). Can be re-run independently to retry a failed send. ~148 lines.
👁️ watchlist-monitor
Anytime — Quick check: Fetches live prices, RSI, SMA position, volume, and last-researched tier for every ticker in watchlist.json + etoro-watchlist.json. No pick generation. Runs in under 3 minutes. Use when you say "how is my watchlist doing?" ~178 lines.
🔄 etoro-watchlist-sync.py — run manually to update
Not a Claude skill — a local Python script. Run it whenever you add or remove instruments in your eToro account.

python3 etoro-watchlist-sync.py --publish

Connects to the eToro public API → pulls all your watchlists → maps eToro numeric instrument IDs to ticker symbols → writes etoro-watchlist.json directly to the Stock-Soup GitHub repo. The hourly skill reads this file during Phase 1 (INIT) — no live eToro API calls happen during automated runs (the eToro API blocks automated requests).
Supporting files:  email-template.html (locked layout) · report-template.md (report structure, on GitHub) · email-builder.py (fills template from JSON) · publish-report.py (pushes to GitHub) · etoro-watchlist-sync.py (manual sync script) · watchlist.json (monitoring tickers, GitHub) · etoro-watchlist.json (eToro positions, GitHub) · watchlist-status.json (per-ticker tier/score, written each run)
Why email-builder.py? Every run that writes its own HTML produces a different layout — breaking formatting for recipients. One locked template, substituted each run via JSON data. The structure is physically impossible to drift between runs.

Watchlists — Three Layers Explained

The system uses three separate watchlist layers. They serve different purposes and are fetched in different ways — none of them automatically override the signal-first discovery process.

1️⃣
Fixed Monitoring List — checked every run
watchlist.json on GitHub · EODHD + Finnhub + smallcaps.com.au

A curated list of tickers stored in watchlist.json in the Stock-Soup GitHub repo — no hardcoded tickers in the skill file. Every run, the system fetches live prices, RSI/SMA technicals, and recent news for each. They enter the scoring pool on equal footing with newly discovered candidates — no automatic pick priority. The watchlist is loaded fresh each run so adding/removing tickers takes effect immediately without editing the skill.

What's fetched each run
✅ Live price via EODHD batch API
✅ RSI + SMA50/200 via Yahoo Finance chart API
✅ Recent news via EODHD news API
✅ Smallcaps.com.au page (RSI, volume, narrative)
✅ SimplyWallSt discount % (daily)
How they become picks
They compete on score — same 0–7+ model as every other candidate.

A watchlist stock with no catalyst scores ~2/5 and won't be picked.
A watchlist stock with a fresh upgrade scores 4.5+ and wins a slot.
To add a ticker to the fixed list: edit watchlist.json in the Stock-Soup GitHub repo. The next hourly run will pick it up automatically — no skill file changes needed.
2️⃣
eToro Watchlist Sync — run manually to update
etoro-watchlist.json on GitHub · refreshed via etoro-watchlist-sync.py --publish

etoro-watchlist-sync.py --publish pulls your eToro watchlists and writes etoro-watchlist.json to the Stock-Soup GitHub repo. The skill loads this file at runtime (Phase 1) — no live eToro API calls happen during hourly runs (the eToro API blocks automated requests).

What it outputs
• Full instrument list per watchlist
• eToro internal instrument ID for each ticker
• Live ask / bid / last price snapshot
• Asset type (Stock, ETF, Forex, Crypto…)
• A ready-to-paste SKILL.md instrument ID table
Why the IDs matter
eToro's API identifies instruments by a numeric ID, not a ticker symbol. The sync script maps e.g. NVDA → 1081 so Claude can call the eToro rates endpoint directly for live prices during each run.
When to re-run it: Whenever you add or remove instruments from your eToro watchlists. Run python3 etoro-watchlist-sync.py locally, then paste the output table into SKILL.md's "Feed 21" section.
3️⃣
Browser Watchlist — personal reminder layer
Saved in browser localStorage · visible on the archive page · not read by Claude

The chip UI on the archive page lets you save tickers locally in your browser. Chips are colour-coded by the tier assigned in the last hourly run (watchlist-status.json written by publish-report.py). These are stored in localStorage — they persist across refreshes but are not read by Claude.

What it does
✅ Visual reminder of tickers you're watching
✅ Survives browser refresh and app restart
✅ Colour-coded by exchange (ASX / US / Global)
✅ Add / remove instantly, no login needed
What it doesn't do
❌ Not sent to Claude automatically
❌ Not fetched for prices or news
❌ Not synced across devices or browsers
❌ Won't influence pick selection
To get Claude to actually research a ticker, use the Request Ticker for Research Queue form — that creates a GitHub issue and gets the ticker added to the Layer 1 fixed list in SKILL.md.

Glossary

ASX
Australian Securities Exchange — where Australian shares trade (opens 10am AEST Mon–Fri)
NYSE / NASDAQ
US stock exchanges — NYSE is traditional blue-chips, NASDAQ is tech-heavy
RSI
Relative Strength Index — 0–100 momentum gauge. Think of it as a speedometer. 70+ = too fast (overbought), 30− = too slow (oversold)
SMA 50 / 200
Simple Moving Average — average price over 50 or 200 days. Being above your SMA50 is like running faster than your usual pace — positive momentum
Short interest
% of shares borrowed and sold by people betting the price will fall. High short interest = many bears. When they're wrong, they all rush to buy at once — that's a squeeze
Short squeeze
When a stock unexpectedly rises — short sellers must buy to close their positions, which pushes the price even higher, fast
Catalyst
The specific event that causes a stock to move: earnings beat, new contract, analyst upgrade, FDA approval, regulatory clearance, insider buy
PRE-MOVE
A catalyst just fired but the stock price hasn't reacted yet. Best entry window — market is slow to price in new information
R:R (Risk:Reward)
How much you could make vs how much you risk. 2.0 = risk $1 to potentially make $2. System requires minimum 1.5 to qualify as a pick
Entry zone
The price range to buy at. Always set as a limit order — never chase with a market order above the zone
Stop loss
The price to exit if wrong. Non-negotiable — set it at the moment you enter the trade
Day pick ⚡
Trade designed to be entered and exited within the same ASX or US session
Week pick 📅
Hold for 1–5 trading days — re-assess at end of week
Long pick 📈
Structural thesis — hold for weeks to months through normal volatility
RISK-OFF ⛔
Market mood where investors sell risky assets and move to cash/gold. BTC dropping 3%+ is the system's trigger
VIX
The "fear index" — measures expected market volatility. Above 25 = fear. Below 15 = calm
AUD/USD
Exchange rate: how many US dollars one Australian dollar buys. Higher AUD = good for importing; lower = good for ASX export stocks
EODHD
End of Day Historical Data — the primary API for live ASX prices and AUD/USD each run
Finnhub
API for US/global stock quotes, analyst consensus ratings, and BTC price (risk-off signal)
FMP
Financial Modeling Prep — provides analyst grades, stock screener, and the economic calendar (run mandatory every session)
Supertrend
A technical indicator that gives a simple UP or DOWN direction signal based on price and volatility bands. Combined with ADX for trend strength
ADX
Average Directional Index — measures trend strength (not direction). Above 25 = strong trend. Below 20 = choppy market, hard to trade
Win rate
Tracked by win-rate.py — the % of past picks that hit their price target vs were stopped out. Visible in Section 15B of every report
